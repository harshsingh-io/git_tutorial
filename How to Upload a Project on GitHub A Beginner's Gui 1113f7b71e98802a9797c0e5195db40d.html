How to Upload a Project on GitHub: A Beginner's Guide

GitHub is a powerful platform for version control and collaboration. It allows developers to share their projects with others around the world, collaborate on code, and manage changes. This beginner-friendly guide will walk you through the process of uploading your project to GitHub step by step.

Prerequisites

Before you begin, ensure you have the following:

  1. A GitHub Account: Sign up at GitHub.com if you haven't already.
  1. Git Installed: Download and install Git from git-scm.com.

Step 1: Create a New Repository on GitHub

  1. Log In to your GitHub account.
  1. On the upper-right corner of any page, click the + icon, then select New repository.
  1. Name your repository (typically the name of your project) and add a brief description.
  1. Choose whether your repository will be Public (anyone can see this repository) or Private (you choose who can see this repository).
  1. Click Create repository.
  1. Initialize this repository, and after doing so, you will see something like this. Just copy the first code snippet if you are starting a new project, or copy the second snippet if you have an existing project on your local system that you want to upload to GitHub.

Step 2: Clone the Repository(Beginner Level)

  1. Open your project in VSCode or Other IDE which you use.
  1. Open terminal of the IDE like I have shown below
  1. Now in the terminal just paste the command which you copied above from the GitHub Website. In my case I am using the first snippet because I have new project in local database and I want to upload it on github.
echo "# git_tutorial" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/harshsingh-io/git_tutorial.git
git push -u origin main
  1. After executing the commands you will get the response something like this.
  1. Now you have follow Step 4 for pushing your existing file on GitHub.

Step 2: Clone the Repository(Intermediate Level)

Cloning creates a copy of your repository on your local computer.

  1. On your repository's page, click the Code button and copy the URL under "Clone with HTTPS".
  1. Open This PC and navigate to the project folder and just copy the path of it just like shown in the image or you can directly do this step in VSCode or other IDE’s terminal.
  1. Open your terminal (Command Prompt on Windows, Terminal on macOS and Linux).
  1. Change the current working directory to where you want the cloned directory. Enter your path in place of “path/to/your/folder” like in my case “C:\Users\harsh\Downloads\AndroidProjects\AppShortcuts”
    cd path/to/your/folder
  1. Type git clone, then paste the URL you copied:
git clone https://github.com/yourusername/yourrepository.git
  1. Press Enter. Your local clone will be created.

Step 3: Add Project Files to the Cloned Repository

  1. Move your project files into the cloned repository directory. If you are using the terminal, you can use the mv command (on macOS and Linux) or move command (on Windows).

Step 4: Track Your Files

  1. Add the files in your local repository. This stages them for the first commit: Alternatively, you can add specific files:
    git add .

    OR :

    git add <filename>

Step 5: Commit the Files

  1. Commit the files that you've staged in your repository: Replace "Add existing file" with a meaningful commit message describing the changes you have made.
    
git commit -m "Add existing file"

Step 6: Push Changes to GitHub

  1. Push the changes in your local repository to GitHub:If you're using a branch other than master, replace master with the branch you want to push to.
    git push -u origin main

Step 7: Verify Upload

  1. Go back to your GitHub repository page, and you should see your files have been uploaded.

Conclusion

Congratulations! You have successfully uploaded your project to GitHub. This platform will now allow you to collaborate with other developers, track the changes made to your project, and revert to previous states if necessary.

FAQs

  • What if I get an error during push?
    • Ensure you have the right access rights and that the repository exists. If you recently initialized your Git settings, it might also be worth checking your configuration settings.
  • How can I collaborate with others on GitHub?
    • To collaborate, you can add collaborators under your repository settings, or others can fork your project, make their changes, and submit a pull request.
  • Can I delete files from my GitHub repository?
    • Yes, you can delete files directly on GitHub or from your local repository and push the changes.

This guide should help you get started with uploading and managing your projects on GitHub. Happy coding!
